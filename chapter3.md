| US08 | Subir comprobante de pago | Como solicitante, quiero subir el comprobante para validar el pedido ante el proveedor. | **Escenario 1:** <br/> Dado que el usuario carga un archivo válido,<br/>Cuando lo envía,<br/>Entonces queda vinculado al pedido.<br/><br/>**Escenario 2:** <br/> Dado que el archivo supera el límite,<br/>Cuando se intenta subir,<br/>Entonces muestra un mensaje de error.<br/><br/> | EP02 |
| US09 | Ver historial de pedidos | Como solicitante, quiero ver mis pedidos anteriores para tener control sobre mi consumo. | **Escenario 1:** <br/> Dado que el usuario accede al historial,<br/>Cuando se listan los pedidos,<br/>Entonces puede ver fecha, tipo y estado.<br/><br/>**Escenario 2:** <br/> Dado que no hay pedidos anteriores,<br/>Cuando se ejecuta la acción,<br/>Entonces se muestra un mensaje indicando que aún no ha realizado pedidos.<br/><br/> | EP02 |
| US10 | Ver pedidos pendientes | Como proveedor, quiero ver todos los pedidos pendientes para analizarlos y tomar acción. | **Escenario 1:** <br/> Dado que el proveedor accede al panel,<br/>Cuando ve los pedidos pendientes,<br/>Entonces puede revisar sus detalles.<br/><br/> | EP03 |
| US11 | Aprobar o rechazar pedido | Como proveedor, quiero aceptar o rechazar pedidos según el stock disponible para evitar conflictos de distribución. | **Escenario 1:** <br/> Dado que el proveedor tiene stock suficiente,<br/>Cuando aprueba el pedido,<br/>Entonces el estado cambia a “Aprobado”.<br/><br/>**Escenario 2:** <br/> Dado que el stock es insuficiente,<br/>Cuando rechaza,<br/>Entonces el estado cambia a “Rechazado” con razón registrada.<br/><br/> | EP03 |
| US12 | Marcar pedido como despachado | Como proveedor, quiero marcar cuándo un pedido sale a entrega para notificar al cliente. | **Escenario 1:** <br/> Dado que el vehículo ha salido,<br/>Cuando el proveedor marca el pedido como despachado,<br/>Entonces el estado cambia.<br/><br/> | EP03 |
| US13 | Cerrar pedido | Como proveedor, quiero cerrar el pedido cuando el cliente confirme la entrega para finalizar el proceso. | **Escenario 1:** <br/> Dado que el pedido ha sido confirmado por el solicitante,<br/>Cuando el proveedor lo cierra,<br/>Entonces ya no puede modificarse.<br/><br/>**Escenario 2:** <br/> Dado que no ha sido confirmado,<br/>Cuando intenta cerrarlo,<br/>Entonces el sistema lo impide.<br/><br/> | EP03 |
| US14 | Generar reporte de ventas | Como proveedor, quiero generar reportes de ventas para tener registro de operaciones realizadas. | **Escenario 1:** <br/> Dado que el proveedor selecciona un rango de fechas,<br/>Cuando solicita el reporte,<br/>Entonces se genera un archivo con los datos.<br/><br/>**Escenario 2:** <br/> Dado que no hay ventas en ese rango,<br/>Cuando genera el reporte,<br/>Entonces se indica que no hay resultados.<br/><br/> | EP03 |
| US15 | Iniciar sesión | Como usuario registrado, quiero iniciar sesión con correo y contraseña para acceder a mi cuenta. | **Escenario 1:** <br/> Dado que los datos son correctos,<br/>Cuando el usuario inicia sesión,<br/>Entonces es redirigido a su dashboard.<br/><br/>**Escenario 2:** <br/> Dado que los datos son incorrectos,<br/>Cuando intenta iniciar sesión,<br/>Entonces se muestra un mensaje de error.<br/><br/> | EP04 |
| US16 | Recuperar contraseña | Como usuario registrado, quiero recuperar mi contraseña para volver a acceder si la olvidé. | **Escenario 1:** <br/> Dado que el usuario ingresa su correo,<br/>Cuando se valida,<br/>Entonces recibe un enlace de recuperación.<br/><br/>**Escenario 2:** <br/> Dado que el correo no existe,<br/>Cuando lo ingresa,<br/>Entonces se le notifica que no se encuentra registrado.<br/><br/> | EP04 |
| US17 | Cerrar sesión | Como usuario, quiero poder cerrar sesión para mantener segura mi cuenta. | **Escenario 1:** <br/> Dado que el usuario está autenticado,<br/>Cuando selecciona cerrar sesión,<br/>Entonces se cierra la sesión actual y se redirige a login.<br/><br/> | EP04 |
| US18 | Ver resumen de pedidos (Solicitante) | Como solicitante, quiero ver un resumen de mis pedidos para identificar cuántos están en proceso o completados. | **Escenario 1:** <br/> Dado que el solicitante entra al dashboard,<br/>Cuando la información se carga,<br/>Entonces visualiza el número de pedidos en los siguientes estados: pendientes, aprobados, despachados, finalizados y rechazados.<br/><br/>**Escenario 2:** <br/> Dado que el solicitante no tiene pedidos registrados,<br/>Cuando se carga el dashboard,<br/>Entonces se muestra un mensaje indicando "No hay pedidos registrados".<br/><br/>**Escenario 3:** <br/> Dado que ocurre un error al cargar el resumen,<br/>Cuando el sistema intenta obtener la información,<br/>Entonces se muestra un mensaje de error e intenta recargar automáticamente.<br/><br/> | EP05 |
| US19 | Ver resumen de pedidos (Proveedor) | Como proveedor, quiero ver un resumen de pedidos gestionados y pendientes para organizar a los clientes. | **Escenario 1:** <br/> Dado que el proveedor entra al dashboard,<br/>Cuando la información se carga,<br/>Entonces visualiza los siguientes KPIs: número de pedidos pendientes, aprobados, rechazados, despachados y finalizados.<br/><br/>**Escenario 2:** <br/> Dado que no hay pedidos en el sistema,<br/>Cuando se carga el dashboard,<br/>Entonces se muestran todos los KPIs con valor cero y un mensaje informativo.<br/><br/>**Escenario 3:** <br/> Dado que ocurre un error de conexión,<br/>Cuando se intenta cargar el resumen,<br/>Entonces el sistema muestra una alerta de error con opción para reintentar.<br/><br/> | EP05 |
| US20 | Endpoint: Login | Como developer, quiero un endpoint POST /api/login para autenticar usuarios. | **Escenario 1:** <br/> Dado que el request contiene credenciales válidas,<br/>Cuando se procesa,<br/>Entonces retorna un token JWT y el status 200.<br/><br/>**Escenario 2:** <br/> Dado que las credenciales no son válidas,<br/>Cuando se procesa,<br/>Entonces retorna status 401 con un mensaje de error.<br/><br/>**Escenario 3:** <br/> Dado que hay un error interno en el servidor,<br/>Cuando se realiza el login,<br/>Entonces retorna status 500 con un mensaje genérico de error.<br/><br/> | EP06 |
| US21 | Endpoint: Recuperar contraseña | Como developer, quiero un endpoint POST /api/password-reset para que permita enviar correo de recuperación. | **Escenario 1:** <br/> Dado que el correo existe en base de datos,<br/>Cuando se envía el request,<br/>Entonces el sistema genera un token y envía el correo de recuperación.<br/><br/>**Escenario 2:** <br/> Dado que el correo no está registrado,<br/>Cuando se envía el request,<br/>Entonces retorna status 404 y no envía ningún correo.<br/><br/>**Escenario 3:** <br/> Dado que hay un fallo en el envío de correo,<br/>Cuando se intenta ejecutar la acción,<br/>Entonces retorna status 500 y se registra el error en los logs.<br/><br/> | EP06 |
| US22 | Endpoint: Logout | Como developer, quiero un endpoint POST /api/logout para cerrar sesión. | **Escenario 1:** <br/> Dado que se envía el token de sesión activo,<br/>Cuando se llama al endpoint,<br/>Entonces la sesión se invalida y retorna status 200.<br/><br/>**Escenario 2:** <br/> Dado que el token ha expirado o es inválido,<br/>Cuando se llama al endpoint,<br/>Entonces retorna status 401 y no realiza ninguna acción.<br/><br/>**Escenario 3:** <br/> Dado que ocurre un error interno,<br/>Cuando se procesa la solicitud de logout,<br/>Entonces retorna status 500 con un mensaje de error.<br/><br/> | EP06 |
| US23 | Endpoint: Crear pedido | Como developer, quiero un endpoint POST /api/pedidos para registrar un nuevo pedido de combustible. | **Escenario 1:** <br/> Dado que se recibe una petición POST válida con datos completos,<br/>Cuando se procesa,<br/>Entonces retorna status 201 con el ID del pedido.<br/><br/>**Escenario 2:** <br/> Dado que faltan campos obligatorios,<br/>Cuando se envía el request,<br/>Entonces retorna status 400 con mensaje de error.<br/><br/> | EP07 |
| US24 | Endpoint: Consultar pedidos por usuario | Como developer, quiero un endpoint GET /api/pedidos?usuario_id=123 para obtener todos los pedidos de un usuario. | **Escenario 1:** <br/> Dado que el usuario tiene pedidos registrados,<br/>Cuando se llama al endpoint con su ID,<br/>Entonces retorna un array con los pedidos.<br/><br/>**Escenario 2:** <br/> Dado que no tiene pedidos,<br/>Cuando se llama el endpoint,<br/>Entonces retorna un array vacío con status 200.<br/><br/> | EP07 |

| US26 | Asignar conductor a pedido | Como proveedor, quiero asignar un conductor para completar la información de despacho. | **Escenario 1:** <br/> Dado que el pedido tiene un vehículo asignado,<br/>Cuando el proveedor selecciona un conductor disponible,<br/>Entonces el sistema vincula al conductor con el pedido.<br/><br/>**Escenario 2:** <br/> Dado que se selecciona un conductor asignado a otro pedido en la misma franja horaria,<br/>Cuando se intenta asignarlo,<br/>Entonces el sistema muestra un mensaje indicando la incompatibilidad.<br/><br/>**Escenario 3:** <br/> Dado que ocurre un error al guardar la asignación,<br/>Cuando el proveedor realiza la acción,<br/>Entonces se muestra un mensaje de error y no se vincula el conductor.<br/><br/> | EP08 |
| US27 | Validar disponibilidad de transporte | Como proveedor, quiero saber qué vehículos están disponibles antes de asignarlos para vincularlos correctamente. | **Escenario 1:** <br/> Dado que el proveedor visualiza el listado de vehículos,<br/>Cuando un vehículo está asignado a otro pedido para la misma fecha y hora estimada,<br/>Entonces el sistema lo muestra como no disponible.<br/><br/>**Escenario 2:** <br/> Dado que un vehículo está disponible,<br/>Cuando el proveedor visualiza el listado,<br/>Entonces el sistema lo presenta como seleccionable.<br/><br/>**Escenario 3:** <br/> Dado que ocurre un conflicto en tiempo real (por ejemplo, se asignó desde otra sesión),<br/>Cuando se intenta seleccionar ese vehículo,<br/>Entonces el sistema bloquea la acción y muestra un mensaje de actualización en tiempo real.<br/><br/> | EP08 |
| US28 | Ver perfil de usuario | Como usuario, quiero ver mis datos de perfil para revisar mi información registrada. | **Escenario 1:** <br/> Dado que el usuario accede a su perfil,<br/>Cuando se carga la vista,<br/>Entonces puede visualizar su nombre, correo y rol.<br/><br/>**Escenario 2:** <br/> Dado que hay un error en la carga del perfil,<br/>Cuando el sistema no puede acceder a los datos,<br/>Entonces muestra un mensaje de error general y sugiere intentar más tarde.<br/><br/>**Escenario 3:** <br/> Dado que el usuario tiene sesión activa,<br/>Cuando se accede al perfil,<br/>Entonces solo se pueden ver los datos del usuario autenticado.<br/><br/> | EP09 |
| US29 | Editar datos de perfil | Como usuario, quiero editar mis datos para mantener mi información actualizada. | **Escenario 1:** <br/> Dado que el usuario modifica uno o más campos del formulario,<br/>Cuando la información ingresada es válida,<br/>Entonces el sistema guarda los cambios correctamente.<br/><br/>**Escenario 2:** <br/> Dado que el usuario deja un campo obligatorio vacío (por ejemplo, el nombre),<br/>Cuando intenta guardar,<br/>Entonces el sistema muestra un mensaje de validación indicando el campo requerido.<br/><br/>**Escenario 3:** <br/> Dado que ocurre un error de servidor al guardar los cambios,<br/>Cuando se realiza la acción,<br/>Entonces el sistema muestra un mensaje de error general sin perder los datos ya ingresados.<br/><br/> | EP09 |
